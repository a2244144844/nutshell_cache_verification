# Cache Unity Test Documents

This directory follows the UCAgent template-style deliverable layout for a Toffee-oriented verification project. It integrates the process documents under `docs/` into a compact submission-facing report set.

## Document Map

| File | Purpose |
| --- | --- |
| `Cache_basic_info.md` | DUT identity, source boundary, interface summary, and tool flow. |
| `Cache_verification_needs_and_plan.md` | Verification goals, coverage strategy, stage plan, and reproducibility plan. |
| `Cache_functions_and_checks.md` | Feature checklist, directed checks, random checks, scoreboard checks, and Toffee coverage mapping. |
| `Cache_line_coverage_analysis.md` | Coverage result summary, functional coverage status, and RTL line coverage limitation. |
| `Cache_bug_analysis.md` | Bug-injection and real RTL bug evidence, detection paths, and recovery notes. |
| `Cache_test_summary.md` | Final result summary, commands, artifacts, risks, and submission readiness. |

Source process records remain in `docs/`. This directory is the integrated report layer.
